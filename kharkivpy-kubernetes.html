Why kubernetes

  • Fast scaling (container based, declarative objects configuration)
  • Fault tolerant (e.g. failures on nodes are not critical)
  • Containers are good abstractions:
    • e.g. local Docker provides similar environment

Why (when) not

  • overall more complex configuration which certainly requires an additional skills
  • for a small application could be overkill as it requires a lot of overhead, e.g. several nodes, masters, infra, other services will eat a lot of CPU/RAM resources.
